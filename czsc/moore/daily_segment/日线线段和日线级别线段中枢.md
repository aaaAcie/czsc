### 日线级别线段+日线级别中枢/本质走势中枢

寻找日线线段与中枢，就是观察**气血（30F线段）在体内生发与凝结的造化：气血顺势冲锋且死守底线，便延展出**主干经络（日线线段）；当气血在某处来回交汇，系统便‘左右寻穴’捕捉其**滞后的能量极值（MA34峰谷）**。若只探得一丝交集，那是**气血初聚的幻相（1段重叠的候选中枢）**，系统会包容这种‘多相并存’继续向右滑动观察；直到某一处真正达成了**三次能量的叠加（3段重叠）**，便犹如结成‘真丹’，瞬间吞噬周边的假象，确立为当下主导的**真脏腑（活跃中枢）**。此后，除非遭遇**异气冲顶（触碰MA170）唤醒历史宿疾，或是走势彻底跌破/突破引力边缘且**杀人见血（3买3卖后破新低/新高质检），旧的因果才会彻底涅槃入库，天地间便又清空重置，开启新一轮的生生不息。

```Python
class Segment30F:
    """30F级别本质线段 (基础经络)"""
    def __init__(self, direction, start_price, end_price, start_index, end_index):
        self.direction = direction      # 1 向上, -1 向下
        self.start_price = start_price
        self.end_price = end_price
        self.start_index = start_index  # 对应全局K线的起始索引
        self.end_index = end_index      # 对应全局K线的结束索引

class CandidateCenter:
    """并行的候选中枢 (一段重叠 或 三段重叠)"""
    def __init__(self, segments, high, low, overlap_type):
        self.segments = segments        # 构成中枢的线段切片 (至少4段)
        self.high = high                # 中枢上轨
        self.low = low                  # 中枢下轨
        self.overlap_type = overlap_type  # 1(BA一段重叠) 或 3(BADC三段重叠/包含临时BA)
        self.is_active = True           # 存活状态

    def check_price_overlap(self, other_center):
        return max(self.low, other_center.low) <= min(self.high, other_center.high)

    def check_segment_overlap(self, other_center):
        """检查两个中枢构成的线段序列是否有交集"""
        a_indices = {s.start_index for s in self.segments}
        b_indices = {s.start_index for s in other_center.segments}
        return bool(a_indices & b_indices)

class DailySegmentEngine:
    def __init__(self):
        # 1. 基础生灭容器
        self.current_segments = []       # 当前正在生长的日线级别线段
        self.completed_segments = []     # 历史确立入库的日线线段
        
        # 2. 脏腑状态机 (中枢管理)
        self.active_center = None        # 当前活跃真中枢 (Type 3)
        self.archived_centers = []       # 历史归档中枢 (等待MA170唤醒)
        self.candidates = []             # 并行孕育的候选中枢队列 (Type 1 或 Type 3)
        
        # 3. 终极打断器 (悬而未决的杀机)
        self.pending_break = None        # 挂起的 3买/3卖 状态字典
        
    # ==========================================
    # 核心事件循环：主干经络运行流
    # ==========================================
    def process_new_30f_segment(self, new_seg, global_ma34, global_ma170):
        """每生成一条新的 30F 线段，触发此主循环"""
        
        # 【第一关】：生死劫验收 (处理挂起的 3买/3卖 雏形)
        if self.pending_break:
            if self._verify_independence(new_seg):
                self.commit_and_reset_system(new_seg, global_ma34, global_ma170)
                return # 破新低/新高，旧时代终结，重新开始
            else:
                self.pending_break = None # 假摔，撤销打断
                
        # 【第二关】：物理防守底线 (顺势破坏)
        if self._is_trend_broken_by(new_seg):
            self.commit_and_reset_system(new_seg, global_ma34, global_ma170)
            return
            
        # 入列：存活确认
        self.current_segments.append(new_seg)

        # 【第三关】：奇经八脉拦截 (MA170 重组 与 逃逸墙)
        if self.active_center and new_seg.direction != self.current_segments[0].direction:
            self._evaluate_rebound_and_escape(new_seg, global_ma170)
            if self.pending_break:
                return # 触发打断雏形，本轮结束，等下一根线段验明正身

        # 【第四关】：播种新希望 (滑动切片，跑四个Gate法则)
        self._try_spawn_new_candidate(global_ma34)

        # 【第五关】：多重宇宙裁决 (更新并行中枢与吞噬)
        self._update_candidates_and_resolve_conflicts(new_seg, global_ma34)

    # ==========================================
    # 内部医理：核心逻辑实现
    # ==========================================
    def _is_trend_broken_by(self, new_seg):
        """实时防守：是否跌破/突破整波行情的起发点"""
        if not self.current_segments: return False
        daily_dir = self.current_segments[0].direction
        start_price = self.current_segments[0].start_price
        
        if daily_dir == 1 and new_seg.direction == -1:
            return new_seg.end_price < start_price
        if daily_dir == -1 and new_seg.direction == 1:
            return new_seg.end_price > start_price
        return False

    def _verify_independence(self, new_seg):
        """独立性质检：杀人见血，必须破新低/新高"""
        target_dir = self.pending_break["expected_direction"]
        extreme_price = self.pending_break["extreme_price"]
        
        if new_seg.direction != target_dir: return False
        
        if target_dir == -1: # 验证3卖
            return new_seg.end_price < extreme_price
        else:                # 验证3买
            return new_seg.end_price > extreme_price

    def _evaluate_rebound_and_escape(self, rebound_seg, global_ma170):
        """评估反抽：MA170特殊重组 与 空间墙碰撞"""
        target_center = self.active_center
        center_A = self.archived_centers[-1] if self.archived_centers else None
        
        current_ma170_val = global_ma170[rebound_seg.end_index] # 提取当下卫气
        
        # 1. MA170 强制复活历史宿疾 A
        if center_A:
            if (rebound_seg.direction == 1 and rebound_seg.end_price >= current_ma170_val) or \
               (rebound_seg.direction == -1 and rebound_seg.end_price <= current_ma170_val):
                target_center = center_A

        # 2. 空间墙碰撞 (生成挂起状态)
        if rebound_seg.direction == 1 and rebound_seg.end_price < target_center.low:
            self.pending_break = {
                "expected_direction": -1, # 下一段必须是向下
                "extreme_price": self.current_segments[-2].end_price # 记录离开中枢的最低点
            }
        elif rebound_seg.direction == -1 and rebound_seg.end_price > target_center.high:
            self.pending_break = {
                "expected_direction": 1,
                "extreme_price": self.current_segments[-2].end_price
            }

    def _update_candidates_and_resolve_conflicts(self, new_seg, ma_array):
        """同位强吞弱：三段重叠强中枢绝对优先吞噬重叠的一段重叠弱中枢"""
        # 生灭裁决 (O(N^2) 扫描)
        actives = [c for c in self.candidates if c.is_active]
        for i, c_A in enumerate(actives):
            for j, c_B in enumerate(actives):
                if i == j: continue
                # 规则：三段重叠优先！只要线段构成有交集，Type 3 绝对吞噬 Type 1，无论先后。
                if c_B.overlap_type == 3 and c_A.overlap_type == 1 and c_A.check_segment_overlap(c_B):
                    c_A.is_active = False

        # 确立主中枢，移交归档
        self.candidates = [c for c in self.candidates if c.is_active]
        for c in self.candidates:
            # 只要是 Type 3 且没有被废弃，就直接成为引力靶子
            if c.overlap_type == 3:
                if self.active_center and self.active_center != c:
                    self.archived_centers.append(self.active_center)
                self.active_center = c

    def _try_spawn_new_candidate(self, ma_array):
        """滑动切片与 Gate 法则寻穴"""
        # 使用最新的五段滑动窗口逻辑
        if len(self.current_segments) < 4: return
        
        # 每次有新线段进来，都可以基于当前整个序列跑一次 find_center
        # (实际实现中可优化为只检测最新组合)
        high, low, status = self.find_center(self.current_segments, ma_array)
        
        if status in ("TEMPORARY", "FINAL"):
            # 只要状态不是 NOT_FOUND 或 FAILED，说明此处有效结丹
            # 根据 find_center 内部的 Gate 4 判定，临时轨(TEMPORARY)也属于达成3段重叠的前置状态
            # 这里简化演示：只要找到，就将其作为 Candidate 添加，具体 overlap_type 由 find_center 返回更佳
            # 假设 find_center 被扩充返回了 overlap_type
            overlap_type = 3 if "Gate 4 验证 MA34 进入 AB" else 1 
            
            # 防止对同一位置的中枢重复添加
            # 检查 candidates 里是否已经存在同一起点或高度重合的中枢组合
            is_duplicate = False
            for c in self.candidates:
                if abs(c.high - high) < 1e-5 and abs(c.low - low) < 1e-5: # 简化判定
                    is_duplicate = True
                    break
                    
            if not is_duplicate:
                # 记录这组有效组合 (至少 4 段)
                valid_segs = self.current_segments[-5:] # 截取关联段
                self.candidates.append(CandidateCenter(valid_segs, high, low, overlap_type))

    # ==========================================
    # 出厂与涅槃：全局质检逻辑
    # ==========================================
    def commit_and_reset_system(self, break_seg, global_ma34, global_ma170):
        """走势终结，触发四大条件出厂质检"""
        segments = self.current_segments
        
        # 质检门：三大必要条件 (至少3段 + 两极锁死 + 营卫交汇)
        if len(segments) >= 3 and \
           self._check_global_trend_relationship(segments) and \
           self._check_ma_cross_correlation(segments, global_ma34, global_ma170):
            
            self.completed_segments.append(segments) # 结丹入库！
        else:
            # 走势未成型/早产，作为废相或延伸处理
            pass 

        # 天地重置，新相方生
        self.current_segments = [break_seg] # 打断段成为新纪元的起点
        self.active_center = None
        self.archived_centers = []
        self.candidates = []
        self.pending_break = None

    def _check_global_trend_relationship(self, segments):
        """两极锁死法则：只看 30F 端点"""
        daily_dir = segments[0].direction
        all_endpoints = []
        for s in segments: all_endpoints.extend([s.start_price, s.end_price])
        
        global_max, global_min = max(all_endpoints), min(all_endpoints)
        start_p, end_p = segments[0].start_price, segments[-1].end_price
        
        if daily_dir == 1:  return start_p == global_min and end_p == global_max
        if daily_dir == -1: return start_p == global_max and end_p == global_min
        return False

    def _check_ma_cross_correlation(self, segments, ma_fast, ma_slow):
        """
        营卫交汇：严格交叉质检 (拒绝同价粘合/飞吻)
        如果未发生真交叉，则说明当前不能成为一根日线级别线段。
        """
        if not segments: return False
        
        start_idx = segments[0].start_index
        end_idx = segments[-1].end_index
        
        # 1. 寻找明确的“初始阵营”
        # 必须跳过开头的同价状态(0)，找到第一个明确的 1 或 -1
        initial_state = 0
        for i in range(start_idx, end_idx + 1):
            if i >= len(ma_fast) or i >= len(ma_slow): break
            
            if ma_fast[i] > ma_slow[i]:
                initial_state = 1
                break
            elif ma_fast[i] < ma_slow[i]:
                initial_state = -1
                break
                
        # 如果整段走势两条均线完全贴合（死水微澜），直接判定未交叉
        if initial_state == 0:
            return False
    
        # 2. 追踪后续走势，寻找“真贯穿”
        for i in range(start_idx, end_idx + 1):
            if i >= len(ma_fast) or i >= len(ma_slow): break
            
            if ma_fast[i] > ma_slow[i]:
                current_state = 1
            elif ma_fast[i] < ma_slow[i]:
                current_state = -1
            else:
                # 【核心】：同价 (ma_fast == ma_slow)，视为飞吻粘合，当前状态置为 0
                current_state = 0 
                
            # 判决：如果当前出现了明确的趋势(非0)，且与初始阵营相反，才是真交叉！
            if current_state != 0 and current_state != initial_state:
                return True
                
        # 走完一圈只经历了 1 -> 0 -> 1，或者一直没变过，皆判定为未交叉
        return False

    # ==========================================
    # 纯数学工具：ABCD 容平拒死极值搜索 (示例寻找极高点)
    # ==========================================
    def _find_peak(self, ma_array, start_idx, end_idx, step):
        """寻找真阳之巅 (容双平，拒三平死水)"""
        for i in range(start_idx, end_idx, step):
            if ma_array[i] >= ma_array[i-1] and ma_array[i] >= ma_array[i+1]:
                if not (ma_array[i-1] == ma_array[i] == ma_array[i+1]):
                    return i, ma_array[i]
        return None, None

    def _calculate_abcd_center(self, last_3_segs, ma_array):
        """编排 ABCD 四穴，返回 High, Low (此处简化调用示意)"""
        idx1, idx3, idx4 = last_3_segs[0].start_index, last_3_segs[1].end_index, last_3_segs[2].end_index
        
        # 1. 找 B点 (最高峰)：左顾右盼
        b_idx, b_val = self._find_peak(ma_array, idx3, idx1, -1)
        if not b_idx: b_idx, b_val = self._find_peak(ma_array, idx3+1, idx4, 1)
        if not b_idx: return None, None
        
        # ... 后续对称寻找 A(局部最低), D(局部最高), C(局部最低) ...
        # 返回 min(b_val, d_val), max(a_val, c_val)
        return 0, 0 # 需助手补全剩余三点逻辑
```

#### 出厂质检

在系统准备把走势打包入库（Commit）之前，加一道“强制质检门（Validation Gate）”，条件不满足就继续等待下一根本质线段。

```Python
def validate_and_commit_daily_segment(self, global_ma34, global_ma170):
    """
    终极出厂质检：在日线线段宣告终结准备入库时，校验四大必要条件。
    """
    segments = self.current_daily_segment
    
    # 【条件一】：至少 30F 线段 3 段
    if len(segments) < 3:
        # 早产儿：级别不够，不能作为日线线段入库。
        # 处理方案：通常标记为“未完成/延伸段”，或者与上一段合并。
        self._handle_abortive_segment()
        return False
        
    # 【条件三】：MA34 和 MA170 的交叉关联
    has_cross_correlation = self._check_ma_cross_correlation(segments, global_ma34, global_ma170)
    if not has_cross_correlation:
        # 营卫不交：能量未发生大级别共振，不能升格为日线线段。
        # 处理方案：降级处理，视为 30F 级别的宽幅震荡。
        self._handle_abortive_segment()
        return False

    # 条件二（顺势）在生长过程中已由 is_trend_broken 保证
    # 条件四（连续）由重置机制保证

    # 全部通过，正式结丹入库！
    self.completed_daily_segments.append(segments)
    return True
```

#### 概念

1. `Segment_30F` (基础经络节点)
- **物理意义：** 市场的最小能量单位（已走完的30分钟本质线段）。
- **必备属性：***`direction`:`1`(向上) 或`-1` (向下)。
    - `start_price`,`end_price`: 起止K线的极端价格。
    - `start_index`,`end_index`: 起止K线在全局数组中的位置（用于切脉找均线）。

2.`Daily_Segment` (主干经络 / 日线线段)

- **物理意义：**由多个`Segment_30F` 首尾相连构成的一波大趋势。
- **生命周期：**从底层`commit_and_reset`开始新生，不断`append` 新的30F线段，直到触发“顺势防守失败”或“3买/3卖独立性质检通过”后宣告死亡（完结入库）。

3.`Candidate_Center` (假包块 / 候选中枢)

- **物理意义：刚刚凑齐3条反向起手的30F线段，通过寻穴算法找到了 A、B、C、D 点，但**目前只有价格重叠，尚未被后续的 MA34 均线穿越。
- **核心状态：**`overlap_type = 1` (代表只有1段重叠)。
- **命运：**存活在`candidate_queue` 中，等待后续走势的喂养。要么升格，要么被同位真中枢吞噬。

4.`Active_Center` (真脏腑 / 活跃中枢)

- **物理意义：**候选中枢在后续的发展中，成功捕获到了 MA34 均线进入其`[low, high]` 区间。能量彻底凝结。
- **核心状态：**`overlap_type = 3` (代表达成了MA34的3段重叠)。
- **作用：** 它是当前行情的“引力中心（靶子）”，所有关于 3买/3卖 的空间逃逸，默认都以它的边界为基准。

5.`Archived_Center` (宿疾 / 归档中枢)

- **物理意义：**当下方（或上方）结出了新的`Active_Center` 且空间不重叠时，旧的主中枢退位，进入历史档案馆。
- **作用：平时静默。一旦走势触发向上的极端反抽并触碰**MA170均线，它将被强制唤醒，重新成为拦截走势的靶子（特殊重组）。

#### 辅助函数 Helper Functions

1. 真诀 1：顺势判定 (`check_global_trend_relationship`)

    **医理：** 将所有 30F 线段的端点扔进一个池子里，提炼出全局的最大值和最小值。只要上行趋势的发源地（第一段起点）等同于全局最小值，且收官地（最后一段终点）等同于全局最大值，这股阳气就是贯通的！

```Python
def check_global_trend_relationship(self, new_segment):
    """
    终极顺势关系校验：两极锁死法则
    只提取 30F 线段的顶底端点，不涉及底层 K 线。
    """
    if not segments:
        return False

    daily_direction = segments[0].direction
    start_point_price = segments[0].start_price # 进入段的起点
    end_point_price = segments[-1].end_price    # 离开段的终点
    
    # 【核心修正】：只搜集 30F 线段的端点价格
    all_endpoints = []
    for seg in segments:

    global_min = min(all_endpoints)

    elif daily_direction == -1:

    return False
```

4. 真诀 4：动态均线重叠监测 (`_check_ma34_3_segments_overlap`)

    **医理：** 测算新生的经络（线段），其内部的气血（MA34）是否流经了候选中枢的脏腑区间。

```Python
def _check_ma34_3_segments_overlap(self, candidate_center, new_segment, ma_array):
    """
    测算在 new_segment 运行的时间段内，是否有任何一个 MA34 的值，
    落入了候选中枢的 [low, high] 区间内。
    """
    start_idx = new_segment.start_index
    end_idx = new_segment.end_index
    
    # 遍历新线段期间的每一根K线的 MA34 值
    for i in range(start_idx, end_idx + 1):
        # 防溢出保护
        if i >= len(ma_array): break 
        
        current_ma = ma_array[i]
        # 如果均线值进入了中枢区间，立刻返回 True (达成3段重叠)
        if candidate_center.low <= current_ma <= candidate_center.high:
            return True
            
    return False
```
5. 真诀 5：寻穴总控与滑动计算 (`find_center`)

    **医理：** 调度之前写好的“左顾右盼、容平拒死”算法，严格按照新的五段滑动窗口和四大 Gate 法则寻找中枢。

```Python
def find_center(self, segments, ma_array):
    """
    按照摩尔缠论的中枢构造滑动寻穴总流程，计算中枢的 [High, Low]。
    segments: 已成型的30F线段列表
    滑动步长 = 2，所有 Gate 失败均向右滑动
    """
    i = 0
    while i + 3 < len(segments):
        seg_12 = segments[i]    # 主趋势段
        seg_23 = segments[i+1]  # 反向段 ← B点唯一来源
        seg_34 = segments[i+2]  # 主趋势段
        seg_45 = segments[i+3]  # 反向段 ← 验证入区
        
        # 尝试获取 seg_56，如果尚未成型则为 None
        seg_56 = segments[i+4] if i + 4 < len(segments) else None

        # Gate 1: 2-3 反向段有没有带来 MA34 峰值（B点）
        b_idx, b_val = self.find_b_point(seg_23, ma_array)
        if b_val is None:
            i += 2; continue          # ← 滑动

        # Gate 2: B点左侧（1-2段）找 A点（必然成功，不滑动）
        a_idx, a_val = self.find_a_point(seg_12, b_idx, ma_array)
        # AB区间临时确立：[a_val, b_val]

        # Gate 3: 4-5 价格严格进入 AB（开区间，同价不算）
        if not (a_val < seg_45.end_price < b_val):
            i += 2; continue          # ← 滑动

        # Gate 4: 4-5 期间的 MA34 有无进入 AB（决定一段还是三段重叠）
        ma34_in_ab = any(a_val < ma_array[k] < b_val
                         for k in range(seg_45.start_index, seg_45.end_index + 1))

        if ma34_in_ab:
            # 三段重叠 → BADC 细化上下轨
            # 如果 seg_56 尚未成型，暂时以 BA 轨道作为临时结果
            if not seg_56:
                return b_val, a_val, "TEMPORARY"
                
            # seg_56 已成型，D 点搜索范围延伸至 seg_56 终止 K 线之前
            d_search_end = seg_56.end_index - 1
            d_idx, d_val = self.find_d_point(b_idx, d_search_end, ma_array)
            if d_val is None: return None, None, "FAILED"
            
            c_idx, c_val = self.find_c_point(b_idx, d_idx, ma_array)
            if c_val is None: return None, None, "FAILED"
            
            center_high = min(b_val, d_val)
            center_low  = max(a_val, c_val)
            
            if center_low >= center_high:
                return None, None, "FAILED"
                
            return center_high, center_low, "FINAL"
        else:
            # 一段重叠 → 直接用 BA
            center_high = b_val
            center_low  = a_val
            return center_high, center_low, "FINAL"

    return None, None, "NOT_FOUND"  # 当前线段序列内未找到中枢
```

---

#### 中枢构造：滑动寻穴总流程

以**下跌趋势**为例，系统对已成型的 30F 线段序列持续执行以下滑动寻穴流程。进入 `higher` 模块的线段均为**已成型**的 30F 线段，每条新线段入列后立即触发检测。

**窗口定义：**

```
seg_12（↓主趋势）→ seg_23（↑反向）→ seg_34（↓主趋势）→ seg_45（↑反向）→ seg_56（↓主趋势）
     1 ─────────── 2 ─────────── 3 ─────────── 4 ─────────── 5 ─────────── 6
```

> **统一滑动规则：Gate 1 或 Gate 3 失败，均将 seg_34 作为新的 seg_12，向右滑动 2 格重试。**

---

##### Gate 1：B点验证（唯一来源：2-3反向段）

- **医理：** 只有**反向线段（2-3）**才有可能让 MA34 发生反向，在均线上留下局部峰值。主趋势段不具备此能力，无需检测。
- **动作：** 在 `seg_23` 的时间范围内（`start_index` 至 `end_index`），以"左顾右盼、容平拒死"法则扫描 MA34，寻找**局部最高点**：
    - 先从 seg_23 的终点（点3）向左倒推扫描至点2，寻找第一个满足条件的峰值。
    - 若左侧未寻得，游标返回点3，向右扫描至点4，继续寻找。
- **B点未找到 → 滑动**（当前 12345 组合无效，seg_34 作为新 seg_12）
- **B点找到 → 确立 `(b_idx, b_val)`，进入 Gate 2**

##### Gate 2：A点确立（B点左侧回溯，必然成功）

- **医理：** 找到阳气极盛的 B点后，向左回溯，在 1-2 主趋势下行段的能量轨迹中，寻找孕育这股阳气的"极寒之地"。
- **动作：** 从 B点 向左倒推扫描至点1（`seg_12.start_index`），寻找**局部最低点**（容平拒死）。
- **A点物理上必然存在**（1-2 是下行段，MA34 必有谷底），记为 `(a_idx, a_val)`。
- **AB区间临时确立：`center_high_tmp = b_val`，`center_low_tmp = a_val`**

##### Gate 3：4-5价格入区验证（严格开区间，4-5成型后触发）

- **检验：** seg_45 的**终点价格**是否严格进入 AB 区间？
    - 判定条件：`a_val < seg_45.end_price < b_val`
    - **同价触线不算进入**（开区间，排除边界）
- **未进入 → 滑动**
- **进入 → 中枢初步成立，以 BA 作为临时上下轨，进入 Gate 4**

##### Gate 4：MA34三段重叠验证（与 Gate 3 同时触发）

- **检验：** seg_45 运行期间，是否有任意一个 MA34 值**严格落入** AB 区间？
    - 判定条件：`any(a_val < ma34[k] < b_val for k in range(seg_45.start_index, seg_45.end_index + 1))`

| 结果 | 含义 | 动作 |
|---|---|---|
| **否** | 一段重叠 | 直接以 **BA** 作为最终中枢上下轨，中枢确立 |
| **是** | 三段重叠 | 触发 **BADC细化**，先以 BA 作临时轨生效，待 D点确立后更新 |

---

#### BADC细化算法（三段重叠时触发）

整个 BADC 算法严格按 **B → A → D → C** 的先后顺序执行。B、A 两点已由 Gate 1/2 确立，此处执行 D、C 两点的寻找，最终覆盖临时 BA 轨道。

1. **寻找 D点（次高峰 - 阳气余波）**

    - **医理：** MA34 具有滞后性——即便 4-5 段结束后价格开始下行，MA34 可能仍在惯性上升，直到 **5-6 段出现转折K线之前**才拐头向下，在 5-6 段内形成次高峰 D点。因此 D点的搜索范围**必须延伸至 5-6 段**，不能止于点4。
    - **游标动作（向右追踪）：**
        - 从 B点 向右正向扫描，**扫描上界为 seg_56 的终止K线之前**（即 `seg_56.end_index - 1`，若 seg_56 已成型）。
        - 寻找继 B点 之后的下一个满足"容平拒死"条件的**局部最高点**，记为 **D点**。
    - **若 seg_56 尚未成型 → 系统以 BA 作为临时上下轨先行生效，等待 seg_56 入列后继续寻找 D点。**

2. **寻找 C点（次低谷 - 阴阳交汇处）**

    - **医理：** B点（最高峰）和 D点（次高峰）之间经历了 3-4 的主趋势下行，MA34 必然形成谷底——**C点物理上必然存在，无需做失败保护。**
    - **游标动作（区间锁定）：**
        - 游标严格限定在 **B点 到 D点 的时间区间内**，从 B点 向右正向扫描至 D点。
        - 寻找区间内 MA34 的**局部最低点**（容平拒死），记为 **C点**。

3. **结丹定界（覆盖临时BA，确立最终中枢区间）**

    四穴齐聚，执行最后的纯数学运算：

    - **最终上轨：** $Center\_High = \min(B\_value, D\_value)$（两峰取低，保证能量在此处被完全压制）
    - **最终下轨：** $Center\_Low = \max(A\_value, C\_value)$（两谷取高，保证能量在此处有绝对支撑）

    只要 $Center\_High > Center\_Low$，临时 BA 轨道被覆盖，中枢区间最终锁定。
