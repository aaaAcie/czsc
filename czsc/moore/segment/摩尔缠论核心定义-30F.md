# 摩尔缠论（Moore CZSC）核心引擎定义与逻辑

> 摩尔缠论是一套完全独立于 CZSC 传统笔/线/段体系的量化分析引擎，
> 全部逻辑基于**绝对价格极值** + **MA5/MA34 物理形态**驱动，
> 不使用包含关系处理、特征序列、传统分型等概念。

---

## 一、数据层对象（`czsc/moore/objects.py`）

### `TurningK` — 转折极值 K 线

| 字段 | 类型 | 含义 |
|------|------|------|
| `raw_bar` | `RawBar` | 对极值 K 线的引用 |
| `mark` | `Mark.G / Mark.D` | 顶（G）或底（D） |
| `price` | `float` | 极值价格（顶用 high，底用 low） |
| `k_index` | `int` | 在原始 K 线序列中的绝对索引 |
| `trigger_k` | `RawBar` | 发出触发信号的"转折 K" |
| `trigger_k_index` | `int` | 触发 K 的绝对索引 |
| `is_valid` | `bool` | 是否通过法则 1、2，顶底正式成立 |
| `is_perfect` | `bool` | **内政**：顶底之间是否存在线段中枢（法则 3，微观几何成立） |
| `is_locked` | `bool` | **外交**：是否已被趋势正式锁定为不可更改的历史锚点（宏观锁定，不可被穿透吞噬） |

> **`is_perfect` 与 `is_locked` 的哲学区别**
> - `is_perfect = True`：这个顶底内部走势"结构完整"，有中枢支撑的反弹/回调
> - `is_locked = True`：这个顶底已被下一段反向走势正式"认证"为历史端点，高维定性完成，任何穿透回溯都不能触碰它

---

### `MooreCenter` — 双轨线段中枢

| 字段 | 含义 |
|------|------|
| `type_name` | `"VISIBLE"`（肉眼可见）或 `"INVISIBLE"`（非肉眼可见） |
| `direction` | 中枢所在线段的延伸方向 |
| `anchor_k0` | 状态机锁定的高纯度锚点 K0 |
| `confirm_k` | 实体反穿 MA5 的确认 K |
| `center_line` | 中枢线价格（确认K的实体价；中枢线即对应轨：上升线段=下轨，下跌线段=上轨） |
| `upper_rail` | 上轨价格 |
| `lower_rail` | 下轨价格 |
| `start_dt / end_dt` | 中枢时间范围 |

---

### `MooreSegment` — 本质线段

由两个连续异向 `TurningK` 构成，内部挂载 `MooreCenter` 列表。
- `direction`：`Up`（底→顶）或 `Down`（顶→底）
- `is_perfect`：由终点 `TurningK.is_perfect` 决定（实线 or 虚线）
- `power`：绝对价格力度 `abs(end_k.price - start_k.price)`

## 二、顶底确立引擎（`_update_turning_k_engine`）

### 2.1 触发机关（转折K）

每一根 K 线进入状态机后，按以下规则判断是否触发转折信号：

**常规触发（找顶时）：**
- `MA5 <= 前一根 MA5`（MA5 停止上涨），且
- `min(open, close) < MA5`（K 线实体有一端穿破 MA5 下方）

**常规触发（找底时）：**
- `MA5 >= 前一根 MA5`（MA5 停止下跌），且
- `max(open, close) > MA5`（K 线实体有一端突破 MA5 上方）

**跳空触发：**
- 实体完全跳空 MA5（向上或向下），无视 MA5 斜率，直接触发

> ⚠️ 转折K 只是"信号灯"，它告诉引擎"此处附近应该产生一个顶/底"，转折K 本身**不是**极值点。

---

### 2.2 极值寻址

触发后，从触发K（不含）向左搜索最近的局部极值（3K 形态：中间 K 高于/低于两侧 K）。  
若无法找到 3K 极值，则取搜索区间内的**绝对极值**作为兜底。

搜索左边界（根据 `turning_ks` 末端方向决定）：
- 末端与新信号**同向**（同向刷新）→ 追溯到上上个异向点之后
- **反向** → 追溯到序列末端之后

**⚠️ 价格比较使用真实极值**：同向刷新的价格过滤，必须用搜索到的真实极值价格（`new_price`）与已确认的旧顶底比较，而**不是**用触发K自身的 `bar.high / bar.low`。触发K只是信号，不是极值。

---

### 2.3 唯一性作废法则（转折K自身是极值时）

**触发条件**：搜索到的极值K 的索引 == 原始信号K 的索引（`extreme_k_index == actual_trigger_index`）

**处理**：
- 转折K（`trigger_k`）自动顺延至其后的**第一根 K 线**（即确认K）
- 极值K 保持不变，仍以搜索到的最高/低点为准
- 目的：转折K 代表"一根线段的新生"，它本身不能是 3K 中间的极值K

```
正常：极值K ←→ 转折K（不同根）
特殊：极值K = = 触发K → 转折K 右移一位 → 极值K ←→ 转折K（相邻）
```

---

### 2.4 候选点管理（`candidate_tk`）

**极值K 与候选K 的关系**：

> - **极值K**（`extreme_bar`）：2.2 搜索完成后得到的那根原始 K 线，强调物理定位——"这根 K 是价格高点/低点所在"。  
> - **候选K**（`new_tk`）：将 `extreme_bar` 包装为 `TurningK(raw_bar=extreme_bar, trigger_k=触发K, ...)` 后进入 `candidate_tk` 托管，强调逻辑状态——"这根 K 正在等待法则验真"。  
> - 两者指向同一根 K 线，只是不同阶段的称呼：极值K 是搜索的输出结果，候选K 是状态机的管理单元。

每次 2.2 搜索完成后，通过 `_can_refresh_candidate(new_extreme_bar, candidate_tk)` 判断：  
本次搜到的新 `extreme_bar`（极值K）能否替换当前 `candidate_tk.raw_bar`（旧候选K 所对应的那根极值K）。

**刷新条件（A OR B 任一满足即可）：**

| 条件 | 参数对比 | 找顶（Mark.G） | 找底（Mark.D） |
|------|----------|---------------|---------------|
| **A（MA5新生）** | `new_extreme_bar.ma5` vs `candidate_tk.raw_bar.ma5` | 新 MA5 > 旧 MA5 | 新 MA5 < 旧 MA5 |
| **B（价格新极值 + 实体重叠）** | `new_price` vs `candidate_tk.price`，实体范围互查 | 新价格 > 旧候选价格 **AND** 新实体下沿 < 旧候选实体上沿 | 新价格 < 旧候选价格 **AND** 新实体上沿 > 旧候选实体下沿 |

> - 条件A（MA5新生）：新极值K 的 MA5 在方向上也创了新极值，说明趋势动能仍在延续，直接替换
> - 条件B（价格破新 + 实体重叠）：价格突破了旧候选，但 MA5 尚未跟上；要求两根 K 的实体有重叠，确认它们在争夺同一价格区域的主导权，才允许替换

**无旧候选时（`candidate_tk is None`）**：直接设置新候选，无需比较条件。

每根 K 线进入时，**优先执行步骤A（轮询校验）**：哪怕没有新触发，已有的 `candidate_tk` 可能在本根 K 线满足了法则 2（金死叉），直接确立。


---

### 2.5 蓝框后移机制

**触发条件**：触发K的实体与**上一个已确认的同向极值K** 的实体有重叠：
- 找顶时：`min(触发K.open, 触发K.close) < max(上一顶.open, 上一顶.close)`
- 找底时：`max(触发K.open, 触发K.close) > min(上一底.open, 上一底.close)`

**注意**：参考点是同向极值（找顶→对比上一个顶；找底→对比上一个底），检测是否"仍身处旧极值实体区间内"。

**处理**：设置 `_waiting_next_as_tk = True`，延后一根 K 线才真正调用 `_process_confirmed_trigger`，让极值 K 先让出位置。

## 三、四法则验真（`_validate_four_rules`）

每次 `candidate_tk` 准备确立时，必须通过以下全部法则：

### 法则 1：3K 极值要求
`tk.raw_bar` 必须是局部极值，即在原始 K 线序列中满足：
- 顶：`curr.high > prev.high AND curr.high > next.high`
- 底：`curr.low < prev.low AND curr.low < next.low`

不满足 → 返回 `(False, False)`，彻底拒绝

### 法则 2：两侧大铡刀（MA5/MA34 金死叉）
在参考点 `ref_tk` 到当前 `tk` 之间，**至少发生一次 MA5 穿越 MA34**（金叉或死叉）。

不满足 → 返回 `(False, False)`，彻底拒绝

> **`ref_tk` 确定方式**：
> - 若上一个确立点与当前 `tk` **同向**（同向刷新场景）→ `ref_tk = turning_ks[-2]`（往前多跳一个）
> - 否则 → `ref_tk = turning_ks[-1]`

### 法则 3：线段中枢包裹
顶底之间是否存在已识别的中枢（`potential_centers` 非空）。
- 满足 → `is_perfect = True`（实线段）
- 不满足 → `is_perfect = False`（虚线段）
- **不拦截线段生成**，仅打标记，供趋势穿透层使用

## 四、趋势穿透层（`_check_penetration` + `_consume_imperfect_chain`）

> 核心哲学：**高维（趋势主权）管死低维（局部结构）**

### 4.1 穿透触发条件（`_check_penetration`）

穿透判定看的是**待被吞噬的中继结构（`turning_ks[-1]`）** 是否稳固，而非新 pivot 自身。

三个穿透级别（OR 递进，高档天然包含低档）：

| Level | 配置值 | 穿透条件 |
|-------|--------|---------|
| 1 | `STRUCT_ONLY` | 中继结构 `is_perfect == False` |
| 2 | `BREAK_SEG_START`（默认） | Level 1 **OR** 新 pivot 价格突破 `segment_start_extreme`（当前线段起点极值） |
| 3 | `BREAK_TREND_GLOBAL` | Level 1 **OR** 新 pivot 价格突破 `trend_high / trend_low`（趋势全局极值） |

```python
engine.penetration_level = 2  # 修改灵敏度
```

### 4.2 多层回溯吞噬（`_consume_imperfect_chain`）

一旦触发穿透，执行 while 循环，**逐层吞噬不稳定结构链**（最多 50 层安全阀）。

每层通过**三重门**：

```
门1（方向门）：turning_ks[-1] 必须为异向点（才是"中继"）
    ↓
门2（双重防御门）：
    ├─ 宏观铁门：is_locked == True → 立即停止（任何级别均生效）
    └─ 弹性微观门：is_perfect == True + Level==1 → 停止（Level2/3 继续看价格）
    ↓
门3（价格门）：新 pivot 价格必须碾压 turning_ks[-2]（旧同向点）
    ↓
✅ 通过 → 吞噬一层（pop 异向中继 + pop 旧同向点，两者存入 ghost_forks）
```

### 4.3 幽灵分叉记录（`ghost_forks`）

每次吞噬时，被消灭的两个点连同其分叉锚点一起存入 `ghost_forks`：

```python
ghost_forks.append((fork_tk, [consumed_same, consumed_opposite]))
# fork_tk：吞噬完成后仍存活于 turning_ks 的锚点（两条路的分叉起点）
# consumed：被消灭的旧同向点 + 旧异向中继（时间顺序排列）
```

图表中以**灰色点划线从锚点出发画出被消灭的枝丫**，直观展示哪些走势被主趋势判定为陷阱。

### 4.4 `is_locked` 打标时机

当第 3 个转折点（如顶2）确立时，意味着 `[顶1 → 底1]` 这条线段已经彻底成为历史定局：
```python
turning_ks[-3].is_locked = True  # 顶1
turning_ks[-2].is_locked = True  # 底1
```

### 4.5 `segment_start_extreme` 更新规则

- **更新时机**：仅当新 `TurningK` 与前一个方向**相反**（真正反向段确立）时更新
    - 新值 = `turning_ks[-2].price`（上一条线段的终点价格，即本段起点）
- **回溯合并期间保持不变**：穿透吞噬发生时，线段是"延申"而不是"重置"，故起点不变

## 五、趋势状态维护（`_update_trend_state` + `_flip_trend`）

### 状态变量

| 变量 | 含义 |
|------|------|
| `trend_state` | 当前趋势方向（`Direction.Up / Down`），初始为 `None` |
| `trend_high` | 趋势内全局最高价 |
| `trend_low` | 趋势内全局最低价 |
| `trend_extreme_k` | 代表全局极值的 `TurningK` |

### 初始化
第一根有效线段（`turning_ks >= 2`）确立后，扫描全部现有顶底初始化趋势方向与极值。

### 极值更新
每次新 `TurningK` 确立后，若价格刷新了对应方向的全局极值，则更新 `trend_high / trend_low` 与 `trend_extreme_k`。

### 趋势翻转双重锁（满足其一即翻转）

**上涨趋势 → 下跌翻转：**
1. **V 型反转**：新底价格 < `trend_low`（直接打穿全局最低）
2. **结构翻转**：新底 `is_perfect == True` 且突破上一线段起点价格

**下跌趋势 → 上涨翻转：**（对称逻辑）

翻转后：`_flip_trend` 重置极值，以翻转触发点为新趋势的起始锚。

## 六、双轨中枢状态机（`_update_center_engine`）

### 运行时机
每根 K 线进入后，**先于**顶底引擎执行（顶底确立前先判断中枢）。

### 两步起手式
**步骤一：抓锚点 K0**
- 当前 K 实体完全站在 MA5 正侧（上涨：`close > MA5`；下跌：`close < MA5`）
- 且未与已有中枢（`potential_centers`）发生价格重叠
- K0 可以滚动刷新（取最新的纯正侧 K）

**步骤二：等待确认 K**
- K0 之后，出现一根实体**反穿 MA5** 的 K 线，即为**确认 K**
- 确认K 的实体价格决定**中枢线**（即中枢的一条轨道）：
  - **双跳空**（confirm_k 与 k0 无价格重叠，且 confirm_k 实体不触碰 MA5）
    - 上升线段：`center_line = max(open, close)`（实体上沿，离 MA5 最近）
    - 下跌线段：`center_line = min(open, close)`（实体下沿，离 MA5 最近）
  - **非双跳空**（实体正常穿越 MA5）
    - 上升线段：`center_line = min(open, close)`（实体下沿，反突破后的价）
    - 下跌线段：`center_line = max(open, close)`（实体上沿，反突破后的价）
- 中枢线即对应轨：**上升线段 = 下轨**，**下跌线段 = 上轨**

### 中枢类型判定
- **肉眼可见中枢**：从 K0 到确认 K 之间，MA5 发生至少 1 次斜率正负交替（折返）
- **非肉眼可见中枢**：无斜率折返，但通过 5K 重叠等物理形态验真

### 中枢定轨
- **肉眼中枢**：一轨为中枢线（confirm_k 实体价）；另一轨为 MA5 第一次折返的峰（上涨）或谷（下跌），无折返时兜底取 K0 的 MA5
- **非肉眼中枢**：一轨为中枢线（confirm_k 实体价）；另一轨寻找 K0 到确认 K 之间与中枢线有价格交集的连续 3K 重叠区

### 中枢向右扩张
生成中枢后，向右延伸直到第一根完全脱离轨道区间的 K 线为止。

### 排他挂载规则
- 新中枢与上一个中枢价格区间**有重叠**时：
  - 肉眼中枢替换非肉眼中枢（肉眼霸权）
  - 双肉眼中枢 → 扩张合并为一个更大的中枢
  - 双非肉眼中枢重叠 → 放弃新中枢

## 七、流水线整体顺序（每根 K 线）

```
update(bar)
    │
    ├─ 1. 喂入 MA5 / MA34 滑窗队列
    │      冷启动保护：MA34 未充盈 → 直接 return
    │
    ├─ 2. _update_center_engine(bar)  ← 先于顶底引擎，采集潜在中枢
    │
    └─ 3. _update_turning_k_engine(bar)
           │
           ├─ [步骤A] 轮询校验 candidate_tk（无需新触发，在每根K上都检验）
           │    ├─ _validate_four_rules(candidate_tk) → (valid, perfect)
           │    ├─ valid=True → 执行穿透判定 → 吞噬 → 确立 → 打 is_locked
           │    └─ return（已确立，本轮结束）
           │
           ├─ [步骤0] 处理上一根触发的蓝框后移确立
           │    └─ _process_confirmed_trigger(当前bar, ...) → 极值搜索 → 刷新candidate
           │
           └─ [主触发] 检测本根 K 线是否触发新的转折信号
                ├─ 优先检测 Refresh（同向刷新）/ 再检测 Reversal（反向转折）
                ├─ 触发 → _process_confirmed_trigger
                │    ├─ 极值搜索（左向3K扫描 + 兜底绝对极值）
                │    ├─ 唯一性作废法则：极值K == 触发K → 转折K右移一位
                │    ├─ 同向刷新价格过滤：用真实极值价格（非bar.high）与旧顶底比较
                │    ├─ _can_refresh_candidate：条件A（MA5新生）或B（价格+实体重叠）
                │    └─ _validate_four_rules → pass → 穿透判定 → 确立
                └─ 蓝框检测：触发K实体与上一同向极值实体重叠 → _waiting_next_as_tk = True
```

## 附录：摩尔缠论量化引擎的"中医心法"映射

这套量化引擎的底层运作，并非简单的数学概率拟合，而是对市场这一"庞大生命体"进行切脉与望诊。这套代码，即是一套完整的电子诊疗系统。

**1. 气血运行：均线系统（MA5 / MA34）**

MA5（卫气）：运行于表，主司防御与极速反应。转折 K 必须由实体反穿 MA5 来触发，这叫"邪气犯表"，是捕捉市场短期异动的最敏锐探针。

MA34（营气）：运行于里，主司底色与滋养。顶底的确立必须伴随 MA5 与 MA34 的金死叉（大铡刀），这代表表皮的伤风感冒已经深入营血，阴阳格局发生了实质性的逆转。

**2. 穴位与骨相：转折 K 与顶底（TurningK）**

转折 K（生穴）：气机刚开始郁结的痛点，随时可能因为新的力量（唯一法则刷新）而转移。

is_perfect（结节）：局部气血淤堵形成的肿块（包裹了线段中枢）。它有抵抗力，能抗住普通的震荡洗盘，但在摧枯拉朽的单边大势面前，依然会被碾碎。

is_locked（骨相）：被宏观大趋势正式册封的历史锚点。肉可烂，气可散，但骨相不可变。只要被 Locked，任何级别的回溯吞噬都不可逾越，它是走势生灭的绝对界碑。

**3. 脏腑成型：本质中枢（MooreCenter）**

中枢是多空双方绞杀最激烈的"中焦脾胃"，是走势蓄能的器官。K0 锚点是实体纯正、未受污染的 K 线，代表多/空方在结营扎寨前的最后一口"纯正真气"。

**4. 生杀大权：趋势穿透与天人合一（Trend Penetration）**

市场的结构是嵌套的，最高维度的"主趋势"拥有绝对的生杀大权。局部的（人）必须绝对服从全局的（天）。

顺势延申（不治之治）：在主趋势未被证伪前，所有未结成完美中枢的反向折腾都是"假动作"。引擎视其为虚线，不予理睬，这叫"任凭风浪起，稳坐钓鱼台"。

回溯吞噬（荡平沉疴）：一旦触发趋势级穿透，主趋势王者归来。引擎的 while 循环将中间所有试图造反的"不完美中枢"和"虚线顶底"连根拔起，在盘面上留下最干净利落的主线段，并以灰色幽灵枝丫记录被消灭的陷阱化石供复盘使用。
