# 摩尔缠论（Moore CZSC）核心引擎定义与逻辑

> 摩尔缠论是一套完全独立于 CZSC 传统笔/线/段体系的量化分析引擎，
> 全部逻辑基于**绝对价格极值** + **MA5/MA34 物理形态**驱动，
> 不使用包含关系处理、特征序列、传统分型等概念。

---

## 一、数据层对象（`czsc/moore/objects.py`）

### `TurningK` — 转折极值 K 线

| 字段 | 类型 | 含义 |
|------|------|------|
| `raw_bar` | `RawBar` | 对极值 K 线的引用 |
| `mark` | `Mark.G / Mark.D` | 顶（G）或底（D） |
| `price` | `float` | 极值价格（顶用 high，底用 low） |
| `k_index` | `int` | 在原始 K 线序列中的绝对索引 |
| `trigger_k` | `RawBar` | 发出触发信号的"转折 K" |
| `trigger_k_index` | `int` | 触发 K 的绝对索引 |
| `is_valid` | `bool` | 是否通过法则 1、2，顶底正式成立 |
| `is_perfect` | `bool` | **内政**：顶底之间是否存在线段中枢（法则 3，微观几何成立） |
| `is_locked` | `bool` | **外交**：是否已被趋势正式锁定为不可更改的历史锚点（宏观锁定，不可被穿透吞噬） |

> **`is_perfect` 与 `is_locked` 的哲学区别**
> - `is_perfect = True`：这个顶底内部走势"结构完整"，有中枢支撑的反弹/回调
> - `is_locked = True`：这个顶底已被下一段反向走势正式"认证"为历史端点，高维定性完成，任何穿透回溯都不能触碰它

---

### `MooreCenter` — 双轨线段中枢

| 字段 | 含义 |
|------|------|
| `type_name` | `"VISIBLE"`（肉眼可见）或 `"INVISIBLE"`（非肉眼可见） |
| `direction` | 中枢所在线段的延伸方向 |
| `anchor_k0` | 状态机锁定的高纯度锚点 K0 |
| `confirm_k` | 实体反穿 MA5 的确认 K |
| `center_line` | 中枢线价格（上升线段=下轨，下跌线段=上轨） |
| `upper_rail / lower_rail` | 上下轨价格（波峰/波谷处可动态升级） |
| `start_dt / end_dt` | 中枢时间范围（`start_dt` 溯源至最左 2K 重叠点） |

---

### `MooreSegment` — 本质线段

由两个连续异向 `TurningK` 构成，内部挂载 `MooreCenter` 列表。
- `direction`：`Up`（底→顶）或 `Down`（顶→底）
- `is_perfect`：由终点 `TurningK.is_perfect` 决定（实线 or 虚线）
- `power`：绝对价格力度 `abs(end_k.price - start_k.price)`

## 二、顶底确立引擎（`_update_turning_k_engine`）

### 2.1 触发机关（转折K）

每一根 K 线进入状态机后，按以下规则判断是否触发转折信号：

**常规触发（找顶时）：**
- `MA5 <= 前一根 MA5`（MA5 停止上涨），且
- `min(open, close) < MA5`（K 线实体有一端穿破 MA5 下方）

**常规触发（找底时）：**
- `MA5 >= 前一根 MA5`（MA5 停止下跌），且
- `max(open, close) > MA5`（K 线实体有一端突破 MA5 上方）

**跳空触发：**
- 实体完全跳空 MA5（向上或向下），无视 MA5 斜率，直接触发

工程裁决顺序（同一根 K）：
- 同时并检“同向刷新”和“反向转折”触发；
- 执行顺序固定为 **同向刷新优先**，若同向成立则本轮结束，否则再尝试反向。

> ⚠️ 转折K 只是"信号灯"，它告诉引擎"此处附近应该产生一个顶/底"，转折K 本身**不是**极值点。

---

### 2.2 极值寻址（转折K驱动）

触发后，按配置选择寻址策略：

- `use_left_3k_locator = True`：从转折K向左扫描，取命中的第一个 3K 形态顶底；
- `use_left_3k_locator = False`：按“先绝对、后局部”的两段式流程：

  1. 先在区间 `[search_start, trigger_index)` 取**绝对极值**（不含触发K）。  
  2. 再看当前段 `[search_start, trigger_index]` 的绝对极值（含触发K）。  
  3. 若二者相等，直接采用步骤1的绝对极值；若不等，则从 `trigger_index` 向左逐个做 3K 扫描，命中的**第一个**作为候选顶底。  
  4. 若左扫未命中，再回退到步骤1的绝对极值兜底。

搜索左边界（根据 `turning_ks` 末端方向决定）：
- 末端与新信号**同向**（同向刷新）→ 追溯到上上个异向点之后
- **反向** → 追溯到序列末端之后，且为避免 3K 重叠，最小起点约束为 `last.k_index + 2`

工程实现补充（与代码一致）：
- Refresh 物理预筛时，会先用 `[last_same_idx + 1, trigger_index]` 这段做一次极值定位。
- 真正确立时若已预定位成功，会直接复用该 `ext_idx`；否则按上面的 `search_start` 规则重算。

**⚠️ 价格比较使用真实极值**：同向刷新的价格过滤，必须用搜索到的真实极值价格（`new_price`）与已确认的旧顶底比较，而**不是**用触发K自身的 `bar.high / bar.low`。触发K只是信号，不是极值。

---

### 2.3 唯一性作废法则（转折K自身是极值时）

**触发条件**：搜索到的极值K 的索引 == 原始信号K 的索引（`extreme_k_index == actual_trigger_index`）

**处理**：
- 转折K（`trigger_k`）自动顺延至其后的**第一根 K 线**（即确认K）
- 极值K 保持不变，仍以搜索到的最高/低点为准
- 目的：转折K 代表"一根线段的新生"，它本身不能是 3K 中间的极值K

```
正常：极值K ←→ 转折K（不同根）
特殊：极值K = = 触发K → 转折K 右移一位 → 极值K ←→ 转折K（相邻）
```

---

### 2.4 候选点管理（`candidate_tk`）

**极值K 与候选K 的关系**：

> - **极值K**（`extreme_bar`）：2.2 搜索完成后得到的那根原始 K 线，强调物理定位——"这根 K 是价格高点/低点所在"。  
> - **候选K**（`new_tk`）：将 `extreme_bar` 包装为 `TurningK(raw_bar=extreme_bar, trigger_k=触发K, ...)` 后进入 `candidate_tk` 托管，强调逻辑状态——"这根 K 正在等待法则验真"。  
> - 两者指向同一根 K 线，只是不同阶段的称呼：极值K 是搜索的输出结果，候选K 是状态机的管理单元。

每次 2.2 搜索完成后，通过 `_is_physically_better(...)` 判断：  
本次新候选是否有资格替换旧候选（或最新同向已确认点）。
其中 `old_scope/new_scope` 的构造与“MA5/价格是否正向刷新”的判定已抽到公共工具 `segment/scope_utils.py`，微观与宏观共用同一口径。

**刷新条件（A OR B 任一满足即可）：**

| 条件 | 参数对比 | 找顶（Mark.G） | 找底（Mark.D） |
|------|----------|---------------|---------------|
| **A（MA5新生）** | `new_scope.ma5` vs `old_scope.ma5` 的趋势极值 | `max(new_scope.ma5) > max(old_scope.ma5)` | `min(new_scope.ma5) < min(old_scope.ma5)` |
| **B（价格更优 + 实体穿透）** | `new_scope` vs `old_scope` 的价格极值 + 实体方向性穿透 | `max(new_scope.high) > max(old_scope.high)` **AND** 新转折K实体上沿 > 旧端点实体下沿 | `min(new_scope.low) < min(old_scope.low)` **AND** 新转折K实体下沿 < 旧端点实体上沿 |

> - 条件A（MA5新生）：统一使用区间 `old_scope/new_scope` 的 MA5 极值刷新判定，不再依赖单点转折K MA5。
> - 条件B（价格+实体穿透）：价格比较也统一改为区间极值刷新，实体比较仍由新转折K对旧端点做方向性穿透。
> - 条件A寻址窗口左边界放宽到 `old_trigger_idx - 1`，允许命中与旧端点同一个 3K 极值。
> - 若同向寻址命中与旧端点同一个 `ext_idx`，则不新建点，只更新旧点的 `trigger_k / trigger_k_index` 为当前触发K。
> - 同向刷新失败时仅废弃新候选，不回退旧端点。

**无旧候选时（`candidate_tk is None`）**：直接设置新候选，无需比较条件。

每根 K 线进入时，**优先执行一次候选校验**：若通过则立即确立；若不通过则立即废弃，不跨 K 线悬置等待。


---

### 2.5 特殊法则（转折K后移一根）

**触发条件**：当次触发流程中，若“转折K本身就是本段极值K”（`extreme_k_index == trigger_k_index`），且该候选点通过四法则验真。

**处理**：
- 当前转折K作废（仅作触发信号，不作为最终转折K）。
- 极值K保持不变。
- 将转折K后移一根：下一根 K 线作为新生转折K，再次调用 `_process_confirmed_trigger` 完成确立。

**核心约束**：该法则只改变 `trigger_k` 的定位，不改变顶底极值搜索结果。

## 三、四法则验真（`_validate_four_rules`）

每次 `candidate_tk` 准备确立时，必须通过以下全部法则：

### 法则 1：3K 极值要求
`tk.raw_bar` 必须是局部极值，即在原始 K 线序列中满足：
- 顶：`curr.high >= prev.high AND curr.high >= next.high`（宽容的平顶法则：允许巨型平顶阵列的存在，取最近的一根极值）
- 底：`curr.low <= prev.low AND curr.low <= next.low`（宽容的平底法则：允许巨型平底阵列的存在，取最近的一根极值）

不满足 → 返回 `(False, False)`，彻底拒绝

### 法则 2：两侧大铡刀（MA5/MA34 金死叉）
在参考点 `ref_tk` 到当前 `tk` 之间，**至少发生一次 MA5 穿越 MA34**（金叉或死叉）。

当前实现放松了扫描边界：  
从“起点3K中间K到终点3K中间K”，扩展为  
“起点3K第一根（`start_idx-1`）到终点3K最后一根（`end_idx+1`）”。

不满足 → 返回 `(False, False)`，彻底拒绝

> **`ref_tk` 确定方式**：
> - 若上一个确立点与当前 `tk` **同向**（同向刷新场景）→ `ref_tk = turning_ks[-2]`（往前多跳一个）
> - 否则 → `ref_tk = turning_ks[-1]`

### 法则 3：结构完整性（级别保障，决定虚实）
引擎会检查 `ref_tk` 到当前 `tk` 之间是否存在已通过“黑K质检”的中枢：
- **实线段** (`is_perfect = True`)：代表该线段内部结成了具备物理支撑的中枢，结构稳固。**或者满足“独立脱离的两K法则”：只要线段的顶峰或谷底，有两根连续的极值 K 线能够完全独立（不沾边地高过或矮过）于该线段内的所有中枢，它就直接获得实线金牌，绝不被判定为内部扰动。**
- **虚线段** (`is_perfect = False`)：虽然满足了极值和金死叉法则，但因内部既缺乏中枢，又未能完成独立脱离，被视为**级别不足**或**结构单薄**。在可视化中以虚线表示。

> ⚠️ **级别保障心法**：
> “中枢成立即线段成立”。若一个波段无法在内部结成包含黑K的中枢，其本质只是主趋势中的一段微观扰动。

### 法则门控开关（valid / perfect）

- `ma34_cross_as_valid_gate = True`（默认）：  
  `is_valid = rule1 and rule2`，`is_perfect = rule3`
- `ma34_cross_as_valid_gate = False`：  
  `is_valid = rule1`，`is_perfect = rule3 and rule2`  
  （即 MA5/MA34 交叉只影响虚实，不阻断顶底成立）

## 四、宏观物理审计层（`_macro_audit_and_replay` & `_check_leap_physics`）

> 核心哲学：**站在未来审判过去，用物理证据抹平中间的噪音陷阱**

### 4.1 双世界与提交快照

现实现采用“微观世界 + 宏观世界”双轨：

- **微观世界**：持续流式生长，允许末端同向刷新（最后一个端点可被替换）。
- **宏观世界**：只做吞噬审计与塌陷，不接受微观末端回写替换。

二者同步方式是 **Committed Snapshot（已提交快照）**：
- 同步源不是全部微观顶底，而是 `micro_turning_ks[:-1]`（最后一个可刷新端点不参与同步）。
- 宏观同步为 **append-only**（只追加不回退），避免出现“宏观端点被微观同向刷新带跑”的问题。
- 宏观吞噬后产生的排除集（`macro_excluded_micro_ids`）会在后续同步中持续生效。
- 同步引擎会为微观点写入/维护 `micro_id`，并在宏观点中保存 `source_micro_id` 映射，保证双世界可追溯。
- 吞噬映射（`macro_swallow_map`）记录“宏观一段吞了哪些微观 ID”，供重建宏观线段与图表提示使用。

### 4.2 审计触发条件与疑点优先级

当某个端点 `Vx` 因结构不完美被打上 `maybe_is_fake=True` 后，不会立刻审判。

审计必须同时满足：
1. 宏观端点数量达到启动门槛（`len(macro_turning_ks) > 4`）；  
2. `Vx` 右侧已积累至少 `audit_link_rounds` 个**已提交宏观点**。

疑点处理策略：
- **老疑点优先**：每轮只取最早的 `maybe_is_fake`；
- **每轮只处理一个疑点**：成功吞噬即结束本轮，等待下一次快照推进后再开新审计。
- `maybe_is_fake` 标记挂在虚线段终点，审计坐标直接使用该终点索引（`idx = fake_idx`）。

### 4.3 深度多层审计与右侧法官池

对被选中的单个疑点 `idx`，左侧锚点按轮次回溯：
1. 第一轮：`start = idx-1`
2. 第二轮：`start = idx-2`
3. ...直到 `audit_link_rounds`

每轮固定一个左锚后，右侧法官池为 `[idx, n]`，并按从近到远尝试；只接受与左锚异向的 `tk_end`。  
并且必须满足“存在实际中间节点可吞噬”（`end_idx > start_idx + 1`），相邻端点不允许判定为成功吞噬。  
这保证“同一锚点内先小吞噬，再大吞噬”；若小范围可解，不拖到下一轮大范围再解。

### 4.4 物理审判（`_check_leap_physics`）

找到合格的候选法官后，必须经过严苛的双轨物理交叉验证：

首先构建两段区间并比较“趋势极值”：
- `old_scope = [seg_start, old_trigger]`（旧法官触发前的趋势区间）
- `new_scope = [seg_start, new_trigger]`（新法官触发前的趋势区间）
- `ma5_is_better` 不再用单点 MA5，而是用区间 MA5 极值是否刷新（上攻看 `max`，下行看 `min`）。
- 价格生长同样使用区间价格极值是否刷新，再叠加实体越界约束，避免单根顶底K的机械比较失真。
- 与微观引擎共用 `segment/scope_utils.py` 的统一刷新判定口径。

**法则分类：**
| 判决法则 | 适用场景 | 通过条件 | 物理意义 |
|---------|---------|---------|----------|
| **法则一（物理边际审判）** | `ma5_is_better == False`（背离/弱势） | `new_scope` 的价格极值刷新 `old_scope`，并且**K线实体层面前后超越**障碍标杆。 | 虽然均线没跟上，但趋势价格已在区间维度完成暴力刺穿，承认其突破！ |
| **法则二（重心与引力双重审判）** | `ma5_is_better == True`（顺势） | 满足上述物理生长的同时，路径内所有 MA5 **不能刺穿**“起点顶底到其转折K区间”的 MA5 趋势极值（上攻看区间最低，下行看区间最高）。 | 允许顶底之后到转折K之间的 MA5 延续生长/衰减，重心锚定更贴近真实趋势发力区。 |

### 4.5 时空坍塌与打入幽灵（Collapse）

一旦找到满足物理法则的法官对某一层起始跳板成功判决：
1. **打入深渊**：将中间被跨越的障碍物们打包扔进 `ghost_forks`（幽灵分叉堆）中。这就是最高维度的清理操作——“历史虚线没有被消除前，不对后续做出误判”。
2. **时空重连**：起跳点 `tk_start` 与 落脚点 `tk_end` 被电焊般连接在一起，成为新世代不可否认的历史粗主干。这终止并收敛了该层级内的所有纠葛。
3. **洗清嫌疑与上钢印**：去掉了终点的虚假标签；对经历考验的后端主干统一赐予 `is_locked = True` 的锁死令，成为后面几轮扫描的不可推倒的石碑。

如果整整拉出了三轮深度扫描都没通过物理审判怎么办？
- 沉默，等待行情推移。当下一次产生新的拐点 $N+2$, $N+3$ 时，这批点将再次充实右侧法官列表，带着比上一代法官更足的能量，向过去的弱点再次下发神谕。

### 4.6 幽灵分叉记录呈现

在最终可视化的图表中，上述被“打入深渊”的中间结构并没有被数据丢弃。它们会保留原本的走势价格，以**灰色点划线从真实的实线锚点处出发，画出一条断头的枝丫**。
这直观地展示了市场在这段时间内制造的诱假动静，是极其珍贵的复盘陷阱化石。

### 4.7 图表展示约定（双世界）

- 微观层：继续展示原始顶底与原始线段（来源世界）。
- 宏观层：展示全部宏观线段（结果世界）。
- 被宏观吞噬覆盖到的微观线段，用灰色弱化显示。
- 宏观吞噬线段保留内部微观 ID 映射，便于复盘“吞噬了哪些微观段”。

## 五、趋势状态维护（`_update_trend_state` + `_flip_trend`）

### 状态变量

| 变量 | 含义 |
|------|------|
| `trend_state` | 当前趋势方向（`Direction.Up / Down`），初始为 `None` |
| `trend_high` | 趋势内全局最高价 |
| `trend_low` | 趋势内全局最低价 |
| `trend_extreme_k` | 代表全局极值的 `TurningK` |

### 初始化
第一根有效线段（`turning_ks >= 2`）确立后，扫描全部现有顶底初始化趋势方向与极值。

### 极值更新
每次新 `TurningK` 确立后，若价格刷新了对应方向的全局极值，则更新 `trend_high / trend_low` 与 `trend_extreme_k`。

### 趋势翻转双重锁（满足其一即翻转）

**上涨趋势 → 下跌翻转：**
1. **V 型反转**：新底价格 < `trend_low`（直接打穿全局最低）
2. **结构翻转**：新底 `is_perfect == True` 且突破上一线段起点价格

**下跌趋势 → 上涨翻转：**（对称逻辑）

翻转后：`_flip_trend` 重置极值，以翻转触发点为新趋势的起始锚。

## 六、双轨中枢状态机（`CenterEngine`）— 观测病房（Pending Center）

### 运行时机
每根 K 线进入后，**先于**顶底引擎执行。

### 核心属性（上帝视角数据总线）
- `center_anchor_idx`: 记录当前中枢发源的**宏观锚点索引**（`candidate_tk` 或 `turning_ks[-1]`）。
- `last_center_end_idx` (**叹息之墙**): 记录上一个已固化中枢的破窗点，绝对禁止新中枢向左穿透此水位线。

### 0. 物理隔离定力（流式单向推进）与备胎 K0 追踪
这是中枢引擎的“底层生灭法则”。中枢引擎不再受宏观策略（顶底分型）的临时改变而回滚，成为纯粹的物理探测仪：
1. **寻求方向及法定起点：** 
   - 仅在 `State 0（空仓寻猎）` 阶段，通过 `_get_macro_anchor` 获取当前大方向。
   - **分水岭隔离**：必须清楚区分 **物理极值点 (Extreme K)** 与 **转折确认点 (Trigger K)**。
     - **Extreme K (山顶/山谷)**：线段的价格边界，仅用于中枢轨道“向左看齐”的搜检极限，处于旧线段时空。
     - **Trigger K (转折确认点)**：反向突破 MA5 且满足 MA5 转折动力学的那一根 K 线。它是新线段的**诞生日**。
   - **法定视界限制**：中枢 K0 的搜索视界**严禁跨越 Trigger K 回溯到历史时空**。所有 K0 必须产生于 Trigger K 及其之后。
   - **转折K特权**：由于 Trigger K 是新时空的拓荒者，它获得 **K0 法定豁免权** —— 即便其成交时的实体未处于 MA5 的“纯洁侧”（is_pure 为 False），它也具备直接确立为 K0 的主权。
2. **物理隔离：** 一旦进入 `State 1` 或 `State 2`，则对当前已被激活的方向绝对忠诚。不再受外界（宏观顶底）的临时虚假信号干预。
3. **合法闭库：** 拥有结界的中枢（闭库），只接受以下两种死法：
   - 外部物理破窗（当前价格的实体完全脱离结界轨道，触发固化）。
   - **内部抢占进化**：
     - **权力视窗**：抢占只能发生在老中枢**彻底形成（名分+黑K通过）之前**。
     - **逻辑**：有同向更优（更高/更低且破开老轨道）的新 K 线触发确认，原地废黜旧种子，重新观测。

### 三段式状态机

| State | 触发条件 | 作用 |
|-------|---------|------|
| **0** | 初始 / rollback 后 | 找 K0：实体**完全**站在 MA5 正侧，且绝不落入前一同向中枢的包含结界内 |
| **1** | K0 就绪 | 找确认K：一旦期间行情漂移回老结界防区，立刻触发回滚。否则等待实体反穿 MA5 成为确认K。 |
| **2** | 确认K 出现 | CENTER_FORMING — 三步流转 |

### State 2 三步流转

**步骤一（入场）：物理发源溯源与极值防火墙**
- **极值防火墙（山顶/山谷隔离）**：以确认K决定一轨，向左看齐寻找另一轨时，**严禁跨越当前线段的物理发源地（Turning Point）**。
- **物理锚定**：搜索视界 `search_start = max(线段锚点, 叹息之墙)`。这确保了中枢作为线段的灵魂，绝不偷取前一线段的价格遗产。

**步骤二（抢占）：胚胎期的权力更迭**
- **抢占视窗**：仅在名分未立或黑K未过（尚未真正成型）的窗口期，备选 K0 才具备篡位资格。
- **抢占法则**：若在观测期间出现同方向更高质量（正向刷新：最高点更高/最低点更低，且破开结界）的新确认K，老中枢当场废黜，原地重建新中枢病房。一旦中枢通过“名分+黑K”双质检，则进入锁定保护期，内部抢占不再生效。

**步骤三（固化 / 脱离）：物理破窗结算与缓冲池机制**
- **脱轨缓冲池（三线脱轨原则）**：当 K 线价格跳出观测结界范围时，系统**不再立刻关闭结界**，而是将该 K 线纳入“脱轨缓冲池”。
  - 若随后的 K 线又跌回结界内，系统判定为假突破/假摔，**清空缓冲池**，中枢继续向右延伸。
  - **只有当连续 3 根 K 线全部站在结界外**，才确立真脱轨，关闭病房进入固化流程。
  - **右边界定格（物理对齐）**：固化时，将时间刻度的右边界（`end_dt`）强制回溯定格在**引发脱轨的第一根 K 线上**。
- **确认K独立性测试（完全脱离检测）**：
  - **定义**：所谓“完全脱离轨道”，是指 K 线的所有价格部分（**包括最高价与最低价**）均处于中枢轨道（Upper/Lower Rail）的外部，且不产生任何重合。
  - **线段实虚判定（法则三）**：一条线段若要被标记为实线（Perfect），其起点山谷或终点山顶必须满足以下任一条件：
    - **双K完全脱离**：端点 K 及其紧邻的一根 K 线，两者的 [Low, High] 区间均完全位于其内部中枢的轨道范围之外。
    - **逻辑意义**：这代表了行情不仅在收盘层面脱离了中枢，在物理波动的每一个刻度上都实现了对中枢“引力场”的绝对逃逸。
- **验证门**：结界固化后回溯内部 K 型进行终极检验 = (起手三式) AND (包含黑K)。
- **隔离守卫（重播干预）**：在重播（Replay）机制下，系统会强制传入当前线段的 `start_ext_idx` (物理极值) 和 `start_trig_idx` (法定起点)，严禁中枢种子在重跑时向左偷步。

### 核心动力学法则

#### 1. 黑K质检（级别之魂）
中枢生命周期内（从确认K之后，到新生中枢线或顶底之前）必须存在至少一根**非正向跳空 MA5** 的 K 线（即实体或影线触碰到 MA5）。
- **目的**：确保多空双方在均线层面有过实质性的阵地交换，保障线段级别。

#### 2. 定性法则：显性/肉眼 (VISIBLE) vs 隐性/非肉眼 (INVISIBLE)
中枢成立后，系统会追踪从 **“确认K起始的正向一笔”** 形成前的 MA5 运行状态，作为分类特征存储在 `is_visible`：
- **VISIBLE**：正向一笔窗口内，MA5 出现过反向运行或走平（斜率回撤）。
- **INVISIBLE**：MA5 一路顺滑递进，无任何折回。

#### 3. 判定方式（起手三式）
不论中枢最终被判定为显性还是隐性，其之所以能成立，必满足以下三种判定图谱的其中之一（存入 `method` 属性）：

| 式 | 名称 | `bars` 时间视界 | 核心逻辑 |
|----|------|--------------|---------|
| 一 | **反正两穿** | from 确认K → 固化点 | K1 反穿中枢线，K2 暴力正穿回去（向上包含或实体越过前高）；双跳空自动成立 |
| 二 | **5K重叠** | from 叹息之墙 → **确认K** | 首K→破坏K→重叠区内 ≥5 根；且**重叠区价格必须覆盖当前中枢线** |
| 三 | **三笔纯势** | from 线段起点 → 固化点 | 极值K→反向3根→正向3根（N字闭环）；阵眼：确认K落在反向一笔内 |

> ⚠️ 三式均不满足 → 关闭病房不予挂载

### 起手三式心法

> 三式互为犄角，合力覆盖所有市场性格，缺一不可：

**V型暴跌/暴涨**：MA5 来不及折返，5K 也没时间重叠，三笔结构尚未成形——此时**式一（反正两穿）** 瞬间将其捕获，一个 K1+K2 的拉锯即宣告中枢成立。

**横盘死皮赖脸**：价格上下乱窜，分不清笔，三笔扫描器迷失——此时**式二（5K重叠）** 用粗暴的阵地战逻辑判定：只要重叠区内挨够了 5 根 K，中枢成立。

**阶梯式阴跌/缓涨回调**：既不暴力（躲过式一），也不横向重叠（躲过式二），走出斜向下/上的 N 字型——此时**式三（三笔纯势）** 启动扫描器，只要确认K落在 N 字型的中间那笔里，当场确诊。

### 终极空间审判与三大延伸法则

这是中枢定型时的“最后一道防线”，其核心为**“是否能与上一个同向中枢完美分离（如上涨线段要求新下轨 > 旧上轨）”**。
完全分离则是一片全新的干净领地（独立挂载新中枢）；未能完全分离，则进入残酷的吞并绞肉机：

| 法则名称 | 触发条件（皆为未完全空间分离） | 处理逻辑 | 核心目的 |
|----------|----------|----------|----------|
| **新皇登基** | **新(肉) vs 老(隐)** | **新中枢为肉眼且老中枢为隐形**时，废黜原有隐形中枢，由新中枢执旗。**注意：肉眼不废肉眼**（若两者皆为肉眼则执行标准轨道合并延伸）。 | 尊重可见结构的最高优先级 |
| **老肉吞新灵** | 老中枢为显形，新中枢为隐形 | 新探测结果作废，新中枢被“招安”，其经历的岁月时间（`end_dt`）由老中枢全额继承。 | 强调已有强结构的稳定性 |
| **灵灵相惜** | 双中枢皆为隐形（非肉眼）相撞 | 保持首个隐形中枢，吸收第二个的时间遗产。 | 防止微观碎波干扰已有结界 |

> ⚠️ **水位线同步（防死循环）**：
> 无论新中枢是挂载还是被吞并，`last_center_end_idx`（叹息之墙）必须同步步进至当前探测器的终点，以确保下一次 $K_0$ 探测从全新的物理位置开始。

### 遗留的“异类幽灵中枢”（Ghost Center）
由于物理时间的不可逆性，当遇到宏观主线段发生“同向极值刷新”时（原先的顶/底被判定为伪顶底，使得包含它的线段被暴力拉伸以吞并这段行情）：
- **物理机制不变**：微观中枢一旦在区间内被客观事实建立，将绝对保留（例如：在红色的上升宏观线段中，包裹着这段时间内由于短暂洗盘所建立出的粉色“向下中枢”残骸）。
- **图表展示**：在可视端，所有这期间方向与最终宏观线段方向相悖的中枢，将被统一定义为 `Ghost Center (👻异类-中枢)`。它将被渲染为**幽灵灰色带虚线框**的幽闭病房形态。这种视觉保留，绝佳地记录了这段历史时间里曾发生的筹码堆积和多空绞杀假象，是对交易体感最深刻的还原。

## 附录：摩尔缠论量化引擎的"中医心法"映射

这套量化引擎的底层运作，并非简单的数学概率拟合，而是对市场这一"庞大生命体"进行切脉与望诊。这套代码，即是一套完整的电子诊疗系统。

**1. 气血运行：均线系统（MA5 / MA34）**

MA5（卫气）：运行于表，主司防御与极速反应。转折 K 必须由实体反穿 MA5 来触发，这叫"邪气犯表"，是捕捉市场短期异动的最敏锐探针。

MA34（营气）：运行于里，主司底色与滋养。顶底的确立必须伴随 MA5 与 MA34 的金死叉（大铡刀），这代表表皮的伤风感冒已经深入营血，阴阳格局发生了实质性的逆转。

**2. 穴位与骨相：转折 K 与顶底（TurningK）**

转折 K（生穴）：气机刚开始郁结的痛点，随时可能因为新的力量（唯一法则刷新）而转移。

is_perfect（结节）：局部气血淤堵形成的肿块（包裹了线段中枢）。它有抵抗力，能抗住普通的震荡洗盘，但在摧枯拉朽的单边大势面前，依然会被碾碎。

is_locked（骨相）：被宏观大趋势正式册封的历史锚点。肉可烂，气可散，但骨相不可变。只要被 Locked，任何级别的回溯吞噬都不可逾越，它是走势生灭的绝对界碑。

**3. 脏腑成型：本质中枢（MooreCenter）**

中枢是多空双方绞杀最激烈的"中焦脾胃"，是走势蓄能的器官。K0 锚点是实体纯正、未受污染的 K 线，代表多/空方在结营扎寨前的最后一口"纯正真气"。

**4. 生杀大权：宏观物理审计与天人合一（Macro Physics Audit）**

市场的结构是嵌套的，最高维度的"主趋势"拥有绝对的生杀大权。但这不再是武断的价格吞并，而是给予了**“滞后审计”的理性**。

顺势延申（不治之治）：在主趋势未被证伪前，所有未结成完美中枢的反向折腾都是"假动作"（被打上 `maybe_is_fake`）。引擎先保持观察的定力，允许虚线存活。

跨越坍塌（荡平沉疴）：一旦最新的走势（法官）证明了之前那段孱弱只是一次彻底的诱导，主趋势立刻王者归来。引擎将发射探针跨越陷阱，一旦通过了严苛的 MA5 均线重心法则与绝对价格法则，原本纠缠交错的结构瞬间发生时空坍塌。中间的一地鸡毛被打包扔进灰色的幽灵碑（Ghost Fork）中。这就是最高维度的物理审判，在盘面上留下的，是最干净利落的主线段，以及见证过历史杀戮的陷阱化石。
